前言

在更换完Hexo的主题后，博客大致就搭建完毕了，这时我们又遇到一个问题，如果我们更换了电脑或者VPS到期了，除了拷贝整个文件夹还有什么好方法可以备份Hexo呢？这次我们就通过git分支功能来备份。

备份步骤

在hexo目录创建本地仓库

添加远程仓库并设置别名为origin

新建并切换到分支hexo

配置忽略文件

查看hexo目录下的.gitignore文件，若无则新建：

该文件会忽略目录中hexo渲染完后生成的public目录和.deploy_git目录

添加hexo目录到本地仓库

推送到远程仓库的hexo分支

第一次push需要用-u关联，之后就不用-u了。
